# Gemini CLI - タスク引き継ぎ資料

## 1. 完了済みタスク: ファイルアップロード機能

### 1.1. 概要
チャット入力欄にファイルアップロード機能を追加し、ユーザーがファイルを添付してメッセージを送信できるようにしました。

### 1.2. 実装済み機能
*   **ファイル選択UI**:
    *   `+`ボタンからファイルを選択し、入力欄の上にプレビューカード（ファイル名、サイズ、削除ボタン）を表示。
    *   ファイル選択の動作安定性を向上済み。
*   **ファイルアップロードAPI**:
    *   バックエンドAPI (`/api/upload`) がファイルを受け取り、タイムスタンプに基づいたユニークなディレクトリに保存。
    *   Next.jsのAPIルートで、アップロード可能なファイルサイズの上限を`1GB`に設定済み。
*   **送信ロジック**:
    *   アップロードが完了すると、ファイルパスとファイル名のリストを含むシステムメッセージを生成し、AIに送信。
*   **キャンセル機能**:
    *   送信ボタンが動的に停止ボタンに変化し、クリックすると進行中のファイルアップロードとAIへのメッセージ送信の両方をキャンセル。
    *   キャンセルされた場合、サーバー上に作成された不完全なディレクトリは自動的に削除される。

## 2. 次のタスク: 添付ファイルのチャットUI改善

### 2.1. 概要
現在、添付されたファイルの情報は`[System]`メッセージとしてユーザーのチャットバブル内にテキストで表示されています。これを改善し、より直感的で分かりやすいUIに変更します。具体的には、システムメッセージを廃止し、ユーザーのメッセージバブルの直上に、添付されたファイルを示すカードを縦に並べて表示します。

### 2.2. 実装計画

#### フェーズ1: データ構造の拡張と通信
*   **目的**: テキストと添付ファイル情報を、それぞれ独立したデータとして扱えるようにする。
*   **`useChat.ts`の修正**:
    *   `sendMessage`関数が、現在の`string`形式だけでなく、`{ text: string, files: FileInfo[] }`という形式のオブジェクトを受け取れるように拡張する。
    *   `FileInfo`は `{ name: string, path: string, size: number }` のような構造とする。
    *   WebSocketでサーバーに送信するJSONメッセージにも、この`files`配列を含めるようにする。
*   **`NewChatPanel.tsx`の修正**:
    *   `handleSendMessage`関数を修正。ファイルアップロード成功後、`[System]`メッセージを生成する代わりに、上記の新しいデータ構造を作成して`sendMessage`関数に渡す。
*   **`server.js`の修正**:
    *   WebSocketサーバーが`files`情報を含むメッセージを受信し、`history`配列に正しく保存できるようにする。

#### フェーズ2: UIレンダリングの変更 (`NewChatPanel.tsx`)
*   **目的**: 拡張されたデータ構造を元に、新しいチャットUIを構築する。
*   **メッセージマッピングロジックの変更**:
    *   `messages.map(...)`内の、ユーザーメッセージ (`msg.role === 'user'`) をレンダリングしている箇所を修正する。
    *   メッセージオブジェクトが`files`プロパティを持っているか確認する。
    *   `files`プロパティが存在する場合、テキストの吹き出しの**上**に、`files`配列をループ処理してファイルカードを**縦に**並べて表示する。
*   **ファイルカードUI**:
    *   各カードには、ファイルアイコン、ファイル名、ファイルサイズなど、プレビューカードと同様の情報を含める。
    *   カードをクリックするとファイルを開く、などのインタラクションは、このフェーズでは実装せず、まずは表示に専念する。

#### フェーズ3: 履歴表示の対応
*   **目的**: 過去の履歴を読み込んだ際も、新しいUIが正しく表示されることを保証する。
*   **動作確認**:
    *   `useChat`フックの履歴読み込み機能 (`fetchHistory`) が、`files`情報を含む過去のメッセージを正しく取得し、UIに反映できることを確認する。
    *   特別な修正は不要な可能性が高いが、念のため、一度チャットを閉じて再度開いた場合や、ブラウザをリロードした場合でも、ファイルカード付きのメッセージが正しく再現されるかを確認する。

## 3. 重要な注意点 / 補足事項
*   この改修は、メッセージのデータ構造というシステムの根幹に関わる変更です。各フェーズで慎重に動作確認を行う必要があります。
*   特に、`useChat.ts`と`server.js`でのデータの一貫性を保つことが重要です。
